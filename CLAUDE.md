# Project Rules

## Git Workflow

- Always create a new branch for every task using git-flow naming: `feat/`, `fix/`, `docs/`, `refactor/`, `chore/`
- Always open a PR after pushing a branch
- Never mention Claude, Claude Code, or AI in commit messages, PR titles, PR descriptions, or code comments
- Never add a "Generated by" footer or attribution to PRs or commits
- Commit messages should be concise and describe the change, not the author
